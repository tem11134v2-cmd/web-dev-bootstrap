# Spec 00.5: Ритуал старта нового проекта (для разработчика)

Эта спека **не для Claude**, а для тебя — разработчика. Проходится один раз на каждый новый сайт, **до** того, как запустить Claude Desktop на проекте. Тут ты создаёшь папку, репо и первый чат Claude.

## Вход

- На Mac установлены: `node` (22+), `npm`/`pnpm`, `git`, `gh` (GitHub CLI, залогинен через `gh auth login`).
- Есть этот bootstrap-репо где-то локально (для копирования).

## 1. Создать папку проекта

```bash
mkdir -p ~/projects/{site}
cd ~/projects/{site}
```

`{site}` — короткое kebab-case имя, оно станет:
- именем папки на Mac (`~/projects/{site}`),
- именем GitHub-репо,
- именем папки на VPS (`~/prod/{site}/`),
- именем PM2-процесса (`{site}-prod`).

Одно имя на всё — меньше путаницы.

## 2. Скопировать bootstrap

```bash
cp -r ~/path/to/web-dev-bootstrap/{CLAUDE.md,docs,specs,.claude,_BUILD} .
```

Или раскатать из template-репо через `gh`:

```bash
gh repo create {owner}/{site} --template tem11134v2-cmd/web-dev-bootstrap --private --clone
cd {site}
```

(Второй вариант удобнее для клиентских проектов, где `{owner}` — аккаунт заказчика.)

## 3. Заполнить `CLAUDE.md`

- Открой `_BUILD/claude-md-template.md` и скопируй содержимое в `CLAUDE.md` поверх текущего (или правь существующий — суть та же).
- Замени `[плейсхолдеры]`: имя проекта, краткое описание, домен, стек-особенности если есть.
- Сохранить.

## 4. Создать репо на GitHub (если не через `gh repo create --template`)

```bash
git init
git add -A
git commit -m "chore: bootstrap"
gh repo create {owner}/{site} --private --source=. --push
```

Для клиентского проекта `{owner}` — аккаунт заказчика (добавит тебя как collaborator). Для своего — твой аккаунт.

**Настрой protected branch `main`:**
- GitHub → Settings → Branches → Add rule → branch name `main` → require PR before merging.
- Пуш напрямую в `main` запрещён. Работаем через ветку `dev` + PR.

## 5. Завести ветку `dev`

```bash
git checkout -b dev
git push -u origin dev
```

С этого момента `main` — только через PR, `dev` — повседневная.

## 6. Открыть в Claude Desktop

- Claude Desktop → New chat → **Select folder** → `~/projects/{site}`.
- Первое сообщение в чате:

  ```
  Read CLAUDE.md and specs/INDEX.md. Then open specs/00-brief.md.
  ```

- Дальше идёшь по спекам 00 → 13.

## 7. Добавить проект в свой реестр (опционально)

Если ведёшь несколько сайтов — полезно иметь один файл `~/projects/_registry.md` с таблицей:

```markdown
| Site   | Repo                  | Domain     | VPS IP         | Start date | Status   |
|--------|-----------------------|------------|----------------|------------|----------|
| alpha  | owner/alpha           | alpha.ru   | 1.2.3.4        | 2026-04-24 | in dev   |
```

Это твой личный реестр — в bootstrap не попадает.

## Что НЕ делаем на этом этапе

- **Не настраиваем сервер.** Это делается в спеке `01b-server-handoff` (после 00, 01a).
- **Не создаём `next-app`.** Это делает Claude в спеке `02-project-init`.
- **Не заполняем `.claude/memory/*`.** Это делает Claude в спеке `00-brief`.

## Готово, когда

- [ ] `~/projects/{site}/` создана, bootstrap-файлы на месте.
- [ ] `CLAUDE.md` заполнен под проект.
- [ ] Репо на GitHub создан, `main` protected.
- [ ] Ветка `dev` создана и запушена.
- [ ] Claude Desktop открыт в папке проекта, первое сообщение отправлено.
