# Examples — образцы реальных спек

Это **не задачи к выполнению** — это живые примеры из реального проекта (migrator.me),
показывающие как выглядит «зрелая» спека. Используй как референс формата:
объём, уровень детализации, баланс «требования / задачи / критерии».

## Что внутри

- `example-20-server-components-refactor.md` — крупный архитектурный рефакторинг
  (перевод 15 страниц из client в server components). Хороший пример как декомпозировать
  сложную задачу на подзадачи и как описать «critical complexity».

- `example-25-performance-audit.md` — финальный performance pass.
  Хороший пример как ссылаться на предыдущие спеки (Background → already done),
  как давать конкретные команды и проверки.

## Как использовать

- Перед написанием своей спеки — открой соответствующий пример, посмотри структуру.
- Шаблон для новой спеки бери из `specs/templates/spec-template.md`, а пример —
  для понимания «как это выглядит когда сделано хорошо».
- НЕ копируй контент примеров в свой проект — там специфика migrator.me.
