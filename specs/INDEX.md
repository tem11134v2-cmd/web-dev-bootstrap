# specs/ — последовательность задач

Спеки — единицы работы для Claude Code. **Одна спека = одна сессия = один коммит-набор**. Запускаются командой:

```
Read and execute specs/04-homepage-and-approval.md.
Before coding:
1. Read the KB files listed in the spec.
2. Show me your implementation plan.
3. Wait for my approval before writing any code.
```

После завершения спеки — `/clear`, открытие следующей. Прыгать через спеки нежелательно: в каждой явно или неявно зашиты артефакты предыдущей.

## Опциональные спеки (`specs/optional/`)

Применяются по требованию проекта, не входят в основной поток.

| Спека | Когда применять | Где встроить |
|---|---|---|
| `opt-quiz.md` | Нужен квиз/калькулятор как лид-магнит | Параллельно 09 или после 06 |
| `opt-ecommerce.md` | Сайт продаёт физические товары (до ~200 SKU без полноценной БД) | Заменяет 05–06 |
| `opt-i18n.md` | Сайт на 2+ языка (next-intl) | После 03, до или вместе с 04 |
| `opt-migrate-from-existing.md` | Есть действующий сайт (Tilda, WP, ...), нужно сохранить SEO + контент | **До или параллельно 00** |

## Правила работы со спеками

- **Plan mode** обязателен в начале каждой спеки — `Shift+Tab×2`. Спека = требования, план — путь реализации.
- **«KB files to read first»** — Claude грузит **только** перечисленные файлы из `docs/`. Не «всё подряд».
- **Один коммит на подзадачу** (см. секцию `Tasks` в каждой спеке). Это save points.
- **Memory updates** в конце каждой спеки — обновить `.claude/memory/project_state.md` (отметить done, указать следующую) и другие файлы по триггерам из CLAUDE.md.
- **Если спека не помещается в одну сессию** (контекст > 60%) — `/clear`, написать handoff.md, продолжить в новой сессии.
- **Никогда не правь спеки задним числом** «как было сделано». Если факт universal — он уходит в `docs/`. Если specific — в `.claude/memory/`. Спека отражает план, не журнал.

## Расширение

Когда возникает задача за пределами 13 основных:

- **Новая страница, блок или статья** — `13-extend-site.md` (циклическая).
- **Полный редизайн** — новая спека `03-design-system v2`.
- **Новая большая фича** (личный кабинет, мультиязычность позже) — отдельная спека или `optional/`.
- **Полный переаудит SEO/perf** — повтор `08-seo-schema` или `11-performance` целиком.

Не путай «расширение» с «новой большой задачей» — для второго пиши отдельную спеку с планом и оценкой, а не размазывай в `13-extend-site`.
