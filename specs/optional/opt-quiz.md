# Spec [opt]: Конверсионный квиз

## Когда применять

В брифе или по ходу проекта возникла идея квиза/калькулятора как лид-магнита.
Типичные сценарии: «подберите свой тип визы», «рассчитайте стоимость переезда», «узнайте подходит ли вам услуга». Хорошо работает для услуг с разветвлённой логикой.

## KB files to read first

- docs/conversion-patterns.md (раздел «Quiz как конверсионный инструмент»)
- docs/forms-and-crm.md (форма-результат отправляется так же как обычный лид)
- `lib/consultation-context.tsx` (если квиз триггерит модалку)
- `app/api/lead/route.ts` (расширим под квиз-данные)

## Goal

Реализовать квиз из 3-7 шагов с прогресс-баром, валидацией, отправкой результата как лид с дополнительными полями (ответы пользователя). На выходе — лиды с квалификацией («что выбрали»), повышенная конверсия за счёт вовлечения.

## Tasks

### 1. Сценарий и контент квиза

1. Получить от пользователя:
   - Список вопросов (3-7 шт.) и варианты ответов на каждый
   - Логика разветвлений (опционально: следующий вопрос зависит от ответа)
   - Финальный экран: что показывать (рекомендация по ответам / форма контакта / благодарность)
2. Записать в `docs/content.md → раздел Quiz`

### 2. Компоненты

3. Создать `components/quiz/QuizWidget.tsx` (client component):
   - State: `currentStep`, `answers`, `isComplete`
   - Прогресс-бар (shadcn `<Progress>`)
   - Кнопки навигации (назад / далее)
   - Финальный шаг — форма контакта (имя + телефон + опц. email)
4. Каждый вопрос как объект:
   ```typescript
   { id: string, question: string, type: 'single' | 'multi' | 'input', options?: { value: string, label: string }[] }
   ```
5. Логика разветвлений (если нужна): функция `getNextStep(currentStep, answers)`

### 3. Интеграция с лид-системой

6. На финальном submit — POST на `/api/lead` с дополнительным полем `quizAnswers: Record<string, string>`
7. В `lib/crm.ts` — мапить `quizAnswers` в кастомные поля CRM (например, в AMO как note или дополнительные поля сделки)
8. Sonner toast: «Спасибо! Мы свяжемся с вами на основании ваших ответов»

### 4. Размещение на сайте

9. Куда вставить квиз:
   - **Hero главной** — как замена/дополнение к стандартному CTA (мощный приём)
   - **Отдельная страница** `/podbor/` или `/calculator/` — если квиз сложный
   - **Final CTA** на странице услуги — альтернатива простой форме
10. Если в hero — сделать через `next/dynamic` (не блокировать LCP):
    ```typescript
    const QuizWidget = dynamic(() => import('@/components/quiz/QuizWidget'), {
      loading: () => <div className="h-[500px] animate-pulse bg-muted/20 rounded-xl" />,
    })
    ```

### 5. Аналитика

11. `trackGoal('quiz_started')` — на первом клике
12. `trackGoal('quiz_step', { step: N })` — на каждом шаге (видим где люди отваливаются)
13. `trackGoal('quiz_completed')` — на финальном submit
14. В Я.Метрика создать соответствующие цели

## Boundaries

- **Always:** прогресс-бар, возможность вернуться назад, валидация на каждом шаге
- **Ask first:** перед добавлением сложной логики разветвлений (часто избыточно), перед персонализированным результатом (требует контента под все комбинации)
- **Never:** запрашивать контакты на первом шаге (теряем людей), делать > 7 шагов (отваливаются)

## Done when

- Квиз работает на localhost: проходится от начала до конца, валидация, прогресс
- Лид с ответами доходит в CRM (с кастомными полями)
- Цели аналитики срабатывают
- Mobile UX корректен (кнопки крупные, шрифт читаемый)
- Lighthouse не упал на странице с квизом (проверить через PSI)
- Деплой выполнен
