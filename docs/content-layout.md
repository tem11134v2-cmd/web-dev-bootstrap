# Content Layout

44 типа секций конверсионных сайтов. Когда пользователь даёт текст — определи тип и используй готовый layout.

## Правила заголовков H2/H3

- Выделять пользу, использовать глаголы действия.
- ❌ «Наши преимущества» → ✅ «Высокая теплоотдача, надёжность и простой монтаж».
- ❌ «Этапы работы» → ✅ «От заявки до результата за 5 шагов».
- **Тест:** человек, читающий **только** H2, понимает суть и преимущества страницы.
- Не начинать с «Наши», «О нашей», «Почему мы».

## Библиотека секций

Формат: `Структура` → `Layout` → `Компоненты shadcn/ui` → `Правила/варианты`.

### Первый экран

**1. Hero.** H1 + подзаголовок (или 3–4 буллета) + 1–2 CTA + карточки преимуществ (3–4). → Full-bleed background, тёмный gradient overlay, текст слева, фото справа. → `Button`. → Варианты: форма вместо CTA, калькулятор/квиз, слайдер офферов, видео-фон. Опция: trust badge, animated counters.

### Доверие

**2. Stats.** 3–6 метрик (крупная цифра + подпись): «12+», «>3000», «98%», «с 2011». → Горизонтальная полоса карточек. → Кастомные counters с `easeOutExpo`.

**3. Logo Bar.** 6–12 логотипов клиентов одного размера (>8 — карусель). → Flexbox row, grayscale → color on hover.

**4. Certificates.** 3–8 сертификатов (изображение + название + год + орган). → Grid карточек. → `Card`, `Dialog` для увеличения.

### Описание и навигация

**5. Service Description.** H2 + вводный абзац + 3–6 пунктов + итог. → Текст + карточки/аккордеон. → `Card` или `Accordion`.

**6. Sub-services.** H2 + 4–8 карточек (иконка + заголовок + описание + ссылка). → Grid 1→2→3-4 col. → `Card`, `Button`. → Если >8 — показать 6–8 + «Все услуги».

**7. Section Navigation.** Для главной: 4–8 карточек-ссылок (иконка + заголовок + описание). → Grid с hover-lift. Самые маржинальные — первыми.

**8. Client Segments.** H2 + 3–6 сегментов (иконка + название + описание). → Grid. → Цель: посетитель узнаёт себя.

**9. Tasks Solved.** H2 + 4–6 пар «задача → решение». → Карточки/таблица/аккордеон.

**10. Triggers.** H2 + 4–7 триггерных ситуаций + CTA «Узнали свою ситуацию?». → Карточки/чек-лист.

**11. What's Included.** H2 + 5–10 пунктов (название + пояснение) + опционально «что НЕ входит». → Чек-лист с иконками.

### Проблема и результат

**12. Problem → Solution.** H2 + 3–5 пар (проблема + решение) + CTA. → Два столбца (красный/серый ↔ зелёный/accent) или карточки. → Проблемы от первого лица клиента, решения конкретные.

**13. Results.** H2 + 4–6 результатов. → Grid карточек с иконками. → Осязаемые: «Экономия 30%», не «Комфорт».

**14. Before/After.** H2 + 3–6 примеров (фото до + после + подпись). → Слайдер с ползунком или парные фото. → Одинаковый ракурс и масштаб.

**15. Consequences.** H2 + 3–5 последствий бездействия + позитивный CTA. → Карточки/таймлайн. → Информировать, не запугивать.

**16. Common Mistakes.** H2 + 4–6 ошибок (ошибка + почему + как правильно). → Аккордеон или карточки.

### Преимущества

**17. Benefits.** H2 + 3–6 карточек. → Grid 1→2→3 col, hover-lift. → Заголовки = выгоды клиента, не характеристики. Не пересекаться со Stats.

**18. Comparison.** H2 + таблица или карточки. Варианты: «мы vs другие», «мы vs самому», «A vs B». → Рекомендуемый вариант выделен. → Не называть конкурентов.

**19. Anti-benefits.** H2 + 3–5 условий «не подходим если» + позитивное завершение. → Цель: фильтрация нецелевых, доверие через честность.

### Цены

**20. Pricing.** Варианты: прайс-таблица / тарифные карточки (2–4 пакета, рекомендованный выделен) / «от X» + факторы / калькулятор. → CTA после цен **обязателен**. → Якорный эффект: средний тариф = рекомендованный.

**21. Promo.** Заголовок + условия + опционально таймер + CTA. → Баннер/карточка с accent фоном. → Конкретная скидка/бонус с **дедлайном**, не «вечные» акции.

### Процесс

**22. Steps.** H2 + 4–7 шагов (номер + заголовок + описание + срок). → Степпер/таймлайн/`Tabs`. → Первый шаг простой («Оставьте заявку»), последний — результат. Сроки диапазоном.

**23. What We Need From You.** H2 + 3–6 пунктов (что нужно + зачем). → Чек-лист.

**24. Timeline.** H2 + 3–6 этапов с длительностью + общий срок. → Горизонтальная диаграмма или список.

### Социальное доказательство

**25. Portfolio.** H2 + 2–4 кейса (фото + задача + что сделали + результат с цифрами). → Карточки/слайдер, лучшие первыми.

**26. Reviews.** H2 + 3–6 отзывов (фото + имя + должность + текст). Варианты: текстовые карточки, скриншоты с площадок, видео, виджет рейтинга. → Отзывы с конкретикой. Виджет рейтинга если ≥ 4.5.

**27. Team.** H2 + 3–8 карточек (фото + имя + должность + описание). → Фото одного стиля. *(Если клиент против — пропустить.)*

**28. About Company (краткий).** H2 + текст + 3–4 ключевых факта + ссылка «Подробнее». → Текст + фото.

**29. Niche Expertise.** H2 + описание + 2–4 примера проектов в нише + метрики. → Текст + карточки.

### География

**30. Map & Contacts.** Карта + адрес + телефон + часы + как добраться. → Два столбца. Карта **lazy load по scroll** (см. `performance.md`). → Телефон/адрес кликабельные на мобильном.

**31. Service Area.** H2 + текст + список городов/районов + карта зоны. → Карта + список в колонках. → Каждый город — потенциальный SEO-запрос. Если >15 — отдельные посадочные.

### FAQ и возражения

**32. FAQ.** H2 + 5–8 пар вопрос-ответ. → `Accordion`, `max-w-3xl`, centered. → **Schema.org FAQPage JSON-LD обязательно** (см. `seo.md`). Вопросы от лица клиента, последний — CTA.

**33. Objections.** H2 + 4–6 возражений (возражение + ответ с гарантией). → `Accordion` или `Card`. → Не ставить рядом с FAQ — объединить или использовать вместо.

**34. Myths.** H2 + 4–6 мифов (миф + реальность). → Карточки с визуальным контрастом.

**35. How to Choose.** H2 + вводный абзац + 4–8 критериев. → `Accordion` или `Card`. → Тон помогающий, не продающий.

**36. Guarantees.** H2 + 3–5 гарантий (заголовок + описание + что при нарушении). → Карточки/баннер. → Конкретные: «Гарантия 5 лет», не «Качество гарантировано».

### SEO-блоки

**37. Tag Cloud.** 6–20 тегов-ссылок на подкатегории/фильтры. → Облако/список в колонках, в нижней части страницы. → Каждый тег — ссылка на страницу с ключевым запросом.

**38. Related.** H2 + 3–6 карточек смежных услуг. → Grid, ближе к концу страницы.

### CTA-блоки

**39. CTA Intermediate.** Заголовок + подзаголовок + кнопка. → Полная ширина, accent/primary фон, centered. → Уникальный текст, max 1–2 на страницу.

**40. CTA Final.** Варианты: форма (2–3 поля), лид-магнит (обложка + форма), квиз, мессенджеры, callback. → Тёмный/accent фон, два столбца. → Минимум полей. Мотиватор + снятие опасения («Не спам», «Бесплатно»).

**41. Popup.** Типы: заявка, exit-intent, акция, лид-магнит, квиз. → `Dialog` по центру. → 1 раз за сессию. Exit-intent только на десктопе. Кнопка закрытия очевидна.

### Товарные блоки (для интернет-магазинов)

**42. Product Card Short.** H1 + краткое описание + 3–5 характеристик + цена + CTA. → `Card`, `Button`, `Badge`.

**43. Product Specs.** Таблица 8–25 строк (параметр + значение), группировка по категориям. → `Table`.

**44. Product Description Extended.** Текст с 2–3 H3 + список преимуществ. → `Tabs` (вкладка «Описание»).
