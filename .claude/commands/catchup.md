---
description: Восстановить контекст проекта по последним изменениям (диф ветки от main)
---

Сделай следующее:

1. `git log main..HEAD --oneline` — посмотри, что в текущей ветке нового vs main. Если ветка == main, переключись: `git log origin/main..HEAD --oneline` или `git log -10 --oneline`.

2. `git diff main..HEAD --stat` — какие файлы изменены и насколько (без полного содержимого).

3. Прочитай `.claude/memory/project_state.md` (только верхние секции до `## Session log`) и сравни с git-историей: совпадает ли `## Active phase` / `## Next steps` с тем, что видно в коммитах?

4. Кратко (5–10 строк) опиши пользователю:
   - **Текущая ветка:** имя
   - **Где по фазе/спеке:** из `## Active phase`
   - **Что в работе:** что коммитили в этой ветке (топ-3 коммита по смыслу)
   - **Открытые задачи:** из `## Next 1-3 steps`

5. Если в коммитах виден явный паттерн (например, все коммиты вида `feat(X)`) — упомяни.

6. **Не лезь в код глубже без отдельного запроса.** Цель `/catchup` — быстрая ориентация, не глубокое погружение. Если пользователь хочет деталей — он попросит явно.
