---
description: Завершить сессию красиво — обновить project_state.md и подготовить hand-off для следующей сессии
---

Сделай следующее, шаг за шагом:

1. Прочитай `.claude/memory/project_state.md`.

2. Собери факты текущей сессии:
   - `git log <session-start-sha>..HEAD --oneline` — какие коммиты сделаны (sha сессии лежит в `/tmp/.claude-session-start-sha`, если файла нет — использовать `git log @{1.hour.ago}..HEAD` как приближение).
   - `git status --porcelain` — есть ли uncommitted-изменения.
   - Проверь, какие файлы тронуты (`git diff --name-only <session-start-sha>..HEAD`).

3. В разделе `## Session log` файла `.claude/memory/project_state.md` **в самом верху** (свежие записи сверху) добавь новую запись:

   ```markdown
   ### Session YYYY-MM-DD HH:MM — короткая суть сессии

   **Done in this session:**
   - [конкретные пункты по коммитам]
   - [файлы, которые тронуты — точные пути]

   **Open at handoff:**
   - [что осталось доделать — конкретно]

   **Uncommitted changes:** нет / есть, список из `git status --porcelain`

   **Resume hint:** [короткая подсказка следующей сессии — 1–2 предложения]

   ---
   ```

   Дату/время бери из `date +"%Y-%m-%d %H:%M"`.

4. Если есть uncommitted-изменения — спроси пользователя: коммитить как часть текущей задачи (атомарно, осмысленным сообщением) или сохранить как есть до следующей сессии. Действуй по ответу.

5. Обнови верхние секции файла под текущую реальность:
   - `## Active phase` — если фаза/спека сменилась
   - `## Active spec` — если перешли на другую спеку
   - `## Blockers` — если появились/ушли блокеры
   - `## Next 1-3 steps` — что планируется в следующей сессии

6. Если завершилась целая фаза/спека — добавь запись в `## Completed phases history` (или `## Completed specs history` для site-проектов) с датой и тегом релиза.

7. Сообщи пользователю одной строкой: «Handoff записан в `.claude/memory/project_state.md`. Можно делать `/clear` — следующая сессия продолжит с этого state через `/resume`.»
