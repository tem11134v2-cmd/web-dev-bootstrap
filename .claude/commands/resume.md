---
description: Продолжить работу в новой сессии — прочитать память, проверить git-state, восстановить контекст
---

Сделай следующее:

1. Прочитай `.claude/memory/INDEX.md`, затем `.claude/memory/project_state.md` целиком.

2. Прочитай **самую свежую** запись в разделе `## Session log` (она сверху).

3. Сравни запись с реальностью репо:
   - `git status --porcelain` — совпадает ли с «Uncommitted changes» из последнего лога?
   - `git log --oneline -10` — соответствуют ли последние коммиты тому, что в «Done in this session»?
   - `git rev-parse HEAD` — на каком коммите мы сейчас?

4. **Если есть расхождение** — стоп, не продолжай. Опиши пользователю что не сходится (например: «в логе сказано 3 коммита, но в git-истории их 5» или «есть uncommitted-изменения, которых нет в логе»). Спроси, как продолжить: довериться текущему состоянию, актуализировать project_state.md или откатить.

5. **Если всё сходится** — кратко (3–5 строк) обобщи пользователю:
   - **Где остановились:** последняя запись «Resume hint» + «Open at handoff»
   - **Активная фаза/спека:** из `## Active phase` / `## Active spec`
   - **Блокеры:** из `## Blockers` (или «нет блокеров»)
   - **Следующие шаги:** из `## Next 1-3 steps`

6. Если в `## Active spec` указан файл спеки (`specs/[XX]-[name].md`) — предложи прочитать его и список «KB files to read first» из неё.

7. **Жди подтверждения от пользователя** перед стартом работы. Не правь код, не запускай билды, не делай коммитов до явного «ОК» / «продолжай».
